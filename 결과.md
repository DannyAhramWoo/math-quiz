## 방식 결정 과정

### 처음 시도 (PDF 브라우저 방식)
- 앱 안에서 PDF 페이지를 넘기며 "이 페이지가 문제 X번" 직접 지정
- PyMuPDF로 PDF 페이지를 이미지로 렌더링
- **단점**: 페이지 설정이 번거롭고 앱이 복잡함

### 최종 채택 (이미지 폴더 방식)
- 문제를 캡처하거나 PDF에서 자동 추출해서 `problems/` 폴더에 저장
- 답을 캡처해서 `answers/` 폴더에 **같은 파일명**으로 저장
- **장점**: 훨씬 빠르고 앱이 단순해짐

## PDF 자동 문제 추출 기능

PyMuPDF + PIL + NumPy로 PDF 페이지를 이미지로 렌더링 후 흰색 여백을 감지해서 자동으로 문제별 분리.

### 실행 방법
```bash
/Users/ahramwoo/miniconda3/bin/python3 - <<'EOF'
import fitz
from PIL import Image
import numpy as np
import io, os

PDF_PATH = "경로/파일명.pdf"
OUT_DIR  = ".../quiz_app/problems"
PREFIX   = "RIPMWC 2022"   # 파일명 앞부분

doc = fitz.open(PDF_PATH)
problem_num = 1

for page_idx in range(len(doc)):
    page = doc[page_idx]
    mat = fitz.Matrix(2.5, 2.5)
    pix = page.get_pixmap(matrix=mat, colorspace=fitz.csRGB)
    img = Image.open(io.BytesIO(pix.tobytes("png"))).convert("RGB")
    arr = np.array(img)
    h = arr.shape[0]
    row_brightness = arr.mean(axis=(1, 2))
    is_white = row_brightness > 245
    search_start = int(h * 0.20)
    search_end   = int(h * 0.80)
    search = is_white[search_start:search_end]
    best_center = None; best_len = 0
    in_gap = False; gap_start = 0
    for j, is_w in enumerate(search):
        if is_w and not in_gap:
            in_gap = True; gap_start = j
        elif not is_w and in_gap:
            gap_len = j - gap_start
            if gap_len > best_len:
                best_len = gap_len
                best_center = search_start + gap_start + gap_len // 2
            in_gap = False
    margin = int(h * 0.015)
    splits = [(margin, best_center), (best_center, h - margin)] if (best_center and best_len > 8) else [(margin, h - margin)]
    for top, bot in splits:
        crop = Image.fromarray(arr[top:bot, :])
        fname = f"{PREFIX}_{problem_num:02d}.png"
        crop.save(os.path.join(OUT_DIR, fname))
        problem_num += 1
doc.close()
EOF
```

## 최종 앱 구조

```
quiz_app/
  app.py              ← Flask 서버
  run.sh              ← 실행 스크립트
  결과.md             ← 이 파일
  templates/
    index.html        ← 웹 UI
  problems/           ← 문제 이미지 (자동 추출 또는 직접 캡처)
  answers/            ← 답 이미지 (같은 파일명)
  progress.json       ← 자동 생성 (진행상황 저장)
```

## 앱 기능

- **🎯 퀴즈**: 순서대로 문제 출제 → 답 보기 → 맞음/틀림 기록
- **📋 목록**: 카테고리별 전체/마스터/틀림/미풀이 통계, 문제 클릭 시 바로 이동
- **❓ 사용법**: 파일명 규칙 및 캡처 방법 안내

### 문제 출제 방식
- 순서대로 출제 (랜덤 아님)
- **맞은 문제**: 마스터 처리 → 퀴즈에서 완전 제외
- **틀린 문제**: 계속 포함
- **미풀이 문제**: 계속 포함
- 목록에서 특정 문제 클릭 → 바로 그 문제로 이동 가능

### 카테고리 분류
- 파일명 자동 분류: `RIPMWC 2022_01.png` → 카테고리 `RIPMWC 2022`, 문제 `01`번
- **서브폴더 자동 분류**: `problems/RIPMWC 2022/01.png` → 카테고리 `RIPMWC 2022`, 문제 `1`번

### 개별 문제 초기화
- 목록 탭에서 각 문제 카드 오른쪽의 **↺** 버튼 클릭 → 해당 문제만 미풀이로 초기화
- 실수로 마스터 처리한 문제 복구 가능

## 사용법

### 1. 이미지 준비
- `Shift + Cmd + 4` 로 문제/답 캡처 (직접 캡처 방법)
- 파일명 규칙: `대회명 년도_문제번호.png`
  - 예: `RIPMWC 2022_01.png`, `SMOPS 2021_15.png`
- `problems/` 폴더와 `answers/` 폴더에 **같은 파일명**으로 저장

### 2. 앱 실행
맥북 켜지면 **자동 시작** (launchd 설정 완료) → 터미널 불필요

수동 관리가 필요할 때:
```bash
# 중지
launchctl unload ~/Library/LaunchAgents/com.ahramwoo.quizapp.plist
# 시작
launchctl load ~/Library/LaunchAgents/com.ahramwoo.quizapp.plist
```

### 3. 브라우저 접속
- 맥북에서: `http://localhost:5001`
- 같은 WiFi 다른 기기: `http://192.168.10.76:5001`
  - IP 바뀌면 터미널에서 `ipconfig getifaddr en0` 확인

> plist 파일: `~/Library/LaunchAgents/com.ahramwoo.quizapp.plist`
> 로그 확인: `cat /tmp/quizapp.error.log`

## Q&A 히스토리

### Q: 문제와 답을 어떻게 분리하고, 틀린 문제만 반복할 수 있는 프로그램을 만들고 싶다
- PDF가 이미지 스캔 방식이라 텍스트 추출 불가
- 이미지 폴더 방식 채택: problems/ + answers/ 폴더 구조

### Q: 이미지 폴더 방식 vs 다른 방식 비교
| 방식 | 장점 | 단점 |
|------|------|------|
| 이미지 폴더 (채택) | 가장 빠르고 단순 | 직접 캡처해야 함 |
| PDF 브라우저 | 캡처 불필요 | 앱에서 페이지 일일이 설정 |
| JSON 수동 설정 | 캡처 불필요 | JSON 파일 직접 편집 |
| OCR 자동 분리 | 완전 자동 | 스캔 품질에 따라 실패 |

### Q: 앱이 "This site can't be reached" 오류
- 서버를 먼저 실행해야 접속 가능 (터미널에서 run.sh 실행)

### Q: VS Code 터미널 vs Mac 터미널
- Mac 기본 터미널 앱 사용 (`Cmd + Space` → "Terminal")
- 퀴즈 앱 사용하는 동안 터미널 창을 열어둬야 함

### Q: 다른 컴퓨터에서도 사용하고 싶다
- 같은 WiFi 방법 채택: 맥북 IP로 접속
- app.py에서 `host='0.0.0.0'`으로 설정 필요
- 맥북 방화벽: Inactive 확인됨

### Q: Claude 메모리가 어디에 저장되나?
- 맥북 로컬: `~/.claude/` 폴더
- OneDrive/Claude 계정 연동 아님
- 이 맥북에서 Claude Code 쓸 때만 기억함

### Q: 카테고리별 분류 (년도/대회별)
- 파일명 자동 분류: `_` 앞부분이 카테고리
- 예: `RIPMWC 2022_01.png` → `RIPMWC 2022` 카테고리

### Q: 문제를 랜덤이 아닌 순서대로
- 순서대로 출제로 변경

### Q: 목록에서 문제 선택해서 바로 이동
- 목록 탭에서 문제 클릭 → 해당 문제로 바로 이동

### Q: 맞은 문제 퀴즈에서 제외, 틀린/미풀이는 계속 포함
- 1회 정답 → 마스터 처리 → 퀴즈 완전 제외
- 틀린 문제 / 미풀이 문제 계속 출제

### Q: PDF에서 문제별로 자동으로 잘라서 저장해줄 수 있나?
- PyMuPDF + PIL + NumPy 사용
- 페이지 내 흰색 여백 감지 → 2분할 자동 저장
- RIPMWC 2021 (25문제), 2022 (20문제), 2023 (18문제) 완료

### Q: problems/answers 폴더 안에 파일이 너무 많다. 서브폴더로 정리할 수 있나?
- `problems/RIPMWC 2022/01.png` 형식으로 서브폴더 지원 추가
- 기존 파일 자동 이동: `RIPMWC 2022_01.png` → `RIPMWC 2022/01.png`
- progress.json 키도 자동 변환하여 기존 진행상황 보존

### Q: 목록에서 문제 클릭하면 항상 1번으로 이동하는 버그
- **원인**: 마스터된 문제는 quizProblems()에 없어서 findIndex가 -1 반환 → index 0(1번)으로 이동
- **수정**: `viewAllMode` 추가 → 목록에서 클릭 시 마스터 포함 전체에서 정확한 위치 탐색
- 카테고리 탭 클릭 시에는 기존 퀴즈 모드(마스터 제외) 유지

### Q: 목록 카드에 문제번호가 "2021번"으로 잘못 표시되는 버그
- **원인**: 파일 이름이 `AP 2021/01.png`로 바뀌었는데 전체 경로에서 첫 번째 숫자(2021)를 매칭
- **수정**: `p.name.split('/').pop()`으로 파일명만 추출 후 숫자 파싱

### Q: 실수로 마스터 처리한 문제를 미풀이로 되돌릴 수 있나?
- 목록 탭 각 카드 우측에 **↺** 초기화 버튼 추가
- 클릭 시 해당 문제만 미풀이로 초기화 (다른 문제 영향 없음)

### Q: 마스터 문제를 목록에서 클릭하면 퀴즈에서 "새 문제"로 잘못 표시되는 버그
- **원인**: 뱃지 표시 로직이 `mastered` 체크 없이 `wrong > 0`만 확인 → 마스터된 문제(wrong=0)가 "새 문제"로 표시
- **수정**: `prog.mastered` 체크를 최우선으로 추가
  ```javascript
  if (prog.mastered) { badge = '마스터 ✅' }
  else if (prog.wrong > 0) { badge = 'N회 틀림' }
  else { badge = '새 문제' }
  ```
- 수정 위치: `index.html` line 280

### Q: PDF 파일에서 문제를 자동으로 잘라서 폴더에 저장해줄 수 있나? (APMOPS 방식)
- **APMOPS 방식**: 녹색 문제번호 원 감지 (PyMuPDF + Pillow + NumPy)
  - 필터 조건: G>100, G-R>30, G-B>30 → 녹색 픽셀 행 군집으로 문제 시작 위치 감지
  - Think Academy 로고는 다색이라 녹색 필터에 안 걸림 → 별도 처리 불필요
  - 페이지당 문제 수 가변 (2~5개), 총 30문제
